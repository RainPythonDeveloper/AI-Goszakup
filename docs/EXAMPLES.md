# Примеры ответов AI-агента

**Проект:** AI-Агент анализа госзакупок РК
**Формат:** 6-блочная структура (Вердикт → Данные → Аналитика → Метод → Уверенность → Рекомендации)
**Языки:** русский (RU), казахский (KZ)

---

## Обзор формата ответа

Каждый ответ AI-агента содержит **6 обязательных блоков:**

| Блок | Название                                           | Содержание                                                                             |
| -------- | ---------------------------------------------------------- | ------------------------------------------------------------------------------------------------ |
| 1        | **Вердикт**                                   | Главный вывод с конкретными цифрами (1-3 предложения)  |
| 2        | **Использованные данные**        | Период, фильтры, источники, размер выборки                    |
| 3        | **Аналитика**                               | Таблицы, числа, медианы, отклонения, ссылки на портал |
| 4        | **Метод оценки**                          | IQR, Isolation Forest, Fair Price, SQL агрегация                                        |
| 5        | **Уверенность и ограничения** | Размер выборки, уровень доверия, fallback-значения            |
| 6        | **Рекомендации**                         | Краткие выводы и действия (1-3 пункта)                               |

## Пример 1: Оценка цены лота (RU)

### Вопрос пользователя

> Оцени адекватность цены лота № 39939053-ОЛ-ЗЦП1

### ReAct-трейс агента

```
Thought: Для оценки цены конкретного лота использую evaluate_lot_price —
         он выполнит всё за один вызов: найдёт лот, рассчитает Fair Price,
         сравнит с аналогами и даст вердикт.
Action:  evaluate_lot_price(lot_number="39939053-ОЛ-ЗЦП1")
Observation: {lot, fair_price, price_assessment, analogues, portal_links}
```

### Ответ агента

**Вердикт:** Цена лота [№ 39939053-ОЛ-ЗЦП1](https://goszakup.gov.kz/ru/announce/index/39939053?tab=lots) (15 000 тг/шт) **незначительно завышена** — отклонение +16,8% от справедливой цены 12 845,30 тг. Цена находится в пределах верхней границы рыночного диапазона (10 200 – 14 500 тг).

**Использованные данные:**

- Период: 15.01.2024 – 12.03.2026
- Фильтры: ENSTRU 441130.000.000001, Алматинская область, единица: Штука
- Источники: оценка цены лота, предметы договоров, расчёт справедливой цены
- Размер выборки: 18 записей, 4 уникальных заказчика

**Аналитика:**

Формула: FairPrice = Медиана × Региональный коэфф. × ИПЦ × Сезонный коэфф.

| Компонент                          | Значение              |
| ------------------------------------------- | ----------------------------- |
| Медиана                              | 12 500,00 тг/шт           |
| Региональный коэфф.        | 1,0215                        |
| ИПЦ (инфляция)                   | 1,0042                        |
| Сезонный коэфф.                | 1,0000                        |
| **Справедливая цена** | **12 845,30 тг/шт** |
| Диапазон (CI)                       | 10 200 – 14 500 тг/шт    |

**Записи для медианы** (контракты, на основе которых рассчитана медиана):

| № | Цена/шт | Дата   | Заказчик                                             | Договор                                                              |
| -- | ------------- | ---------- | ------------------------------------------------------------ | --------------------------------------------------------------------------- |
| 1  | 10 200 тг   | 12.03.2024 | ГУ «Управление здравоохранения» | [Договор](https://goszakup.gov.kz/ru/egzcontract/cpublic/show/14523781) |
| 2  | 11 800 тг   | 05.06.2024 | ТОО «МедСнаб»                                    | [Договор](https://goszakup.gov.kz/ru/egzcontract/cpublic/show/14891234) |
| 3  | 12 500 тг   | 18.09.2024 | ГУ «Больница №7»                                | [Договор](https://goszakup.gov.kz/ru/egzcontract/cpublic/show/15234567) |

**Аналоги** (контракты других организаций):

| № | Цена/шт | Дата   | Заказчик                      | Договор                                                              |
| -- | ------------- | ---------- | ------------------------------------- | --------------------------------------------------------------------------- |
| 1  | 11 500 тг   | 22.04.2025 | КГП «Поликлиника №3» | [Договор](https://goszakup.gov.kz/ru/egzcontract/cpublic/show/16789012) |
| 2  | 13 200 тг   | 10.07.2025 | ГУ «ДСМР Алматы»        | [Договор](https://goszakup.gov.kz/ru/egzcontract/cpublic/show/17012345) |
| 3  | 14 500 тг   | 03.11.2025 | ТОО «Фарм-Центр»        | [Договор](https://goszakup.gov.kz/ru/egzcontract/cpublic/show/17345678) |

Уникальных заказчиков в аналогах: 3.

**Метод оценки:** Справедливая цена = Медиана × Региональный коэффициент × ИПЦ × Сезонный коэффициент. Медиана рассчитана из 18 фактических контрактов (PERCENTILE_CONT 0.5).

**Уверенность и ограничения:**

- Размер выборки: 18 записей → уровень доверия **средний** (10-29)
- Сезонный коэффициент не рассчитан (недостаточно данных: нужно >= 5 записей в квартале и >= 20 всего), использовано значение 1,0
- 4 уникальных заказчика — репрезентативность приемлемая

**Рекомендации:**

1. Цена лота находится в пределах рыночного диапазона (верхний квартиль). Рекомендуется запросить обоснование цены у заказчика.
2. При увеличении выборки (>= 30 записей) уровень доверия повысится до «высокого».

## Пример 2: Ценовые аномалии (RU)

> Найди закупки с подозрительно высокими ценами

Action:  detect_anomalies(method="consensus", threshold=1.5)

## Пример 3: Аналитический запрос (RU)

### Вопрос пользователя

> Покажи топ-5 заказчиков по общей сумме договоров

### ReAct-трейс агента

```
Thought: Для рейтинга заказчиков нужен SQL с GROUP BY, SUM и JOIN
         на subjects для имён организаций.
Action:  execute_sql("SELECT c.customer_bin, s.name_ru,
         COUNT(*) as contract_count,
         SUM(c.contract_sum) as total_sum
         FROM contracts c
         LEFT JOIN subjects s ON c.customer_bin = s.bin
         WHERE c.contract_sum > 0
         GROUP BY c.customer_bin, s.name_ru
         ORDER BY total_sum DESC LIMIT 5")
Observation: 5 rows returned
```

### Ответ агента

**Вердикт:** Топ-5 заказчиков за 2024-2026 гг. суммарно заключили договоров на 287,3 млрд тг, что составляет 68% от общей суммы всех договоров в базе.

**Использованные данные:**

- Период: 2024-2026 гг.
- Фильтры: все заказчики (27 организаций), сумма > 0
- Источники: SQL-запрос к базе данных (договоры, организации)
- Размер выборки: 32 574 договора

**Аналитика:**

| № | Заказчик                                                            | БИН       | Кол-во договоров | Сумма (тг) | Доля |
| -- | --------------------------------------------------------------------------- | ------------ | ------------------------------ | ----------------- | -------- |
| 1  | ГУ «Департамент полиции г. Астаны»             | 000740001307 | 4 521                          | 98 450 000 000    | 23,3%    |
| 2  | КГП «Городская клиническая больница №1»   | 050740004819 | 3 876                          | 67 230 000 000    | 15,9%    |
| 3  | ГУ «Управление образования ВКО»                 | 210240019348 | 2 934                          | 52 180 000 000    | 12,4%    |
| 4  | РГП «Казахстан инжиниринг»                          | 780140000023 | 2 156                          | 38 900 000 000    | 9,2%     |
| 5  | ГУ «Департамент здравоохранения Алматы» | 140340016539 | 1 823                          | 30 540 000 000    | 7,2%     |

**Метод оценки:** SQL агрегация (GROUP BY customer_bin, SUM, COUNT) с сортировкой по убыванию суммы.

**Уверенность и ограничения:**

- Размер выборки: 32 574 договора → уровень доверия **высокий**
- В выборку включены все договоры с ненулевой суммой

**Рекомендации:**

1. Для топ-3 заказчиков рекомендуется проверить концентрацию поставщиков (не доминирует ли один поставщик).
2. Высокая доля (23,3%) первого заказчика может быть обусловлена спецификой закупок — рекомендуется детализация по категориям ТРУ.

## Пример 4: Концентрация поставщиков (RU)

### Вопрос пользователя

> Найди заказчиков с высокой концентрацией поставщиков

### ReAct-трейс агента

```
Thought: Для выявления концентрации использую check_supplier_concentration
         с порогом 80% (по умолчанию).
Action:  check_supplier_concentration(threshold_pct=80)
Observation: {total: 8, alerts: [...]}
```

### Ответ агента

**Вердикт:** Обнаружено **8 случаев** высокой концентрации поставщиков, где один поставщик получает более 80% от общей суммы закупок заказчика. Из них 2 — критических (доля > 95%).

**Использованные данные:**

- Период: 2024-2026 гг.
- Фильтры: все заказчики, порог 80%
- Источники: анализ концентрации поставщиков
- Размер выборки: 27 организаций

**Аналитика:**

🔴 **Critical** (доля > 95%):

| Заказчик                                                              | Поставщик                                      | Доля | Сумма поставщика (тг) |
| ----------------------------------------------------------------------------- | ------------------------------------------------------- | -------- | -------------------------------------- |
| ГУ «Управление ЖКХ Караганды» (БИН 100140011059) | ТОО «СтройСервис» (БИН 940740000911) | 97,3%    | 2 450 000 000                          |
| КГП «Поликлиника №12» (БИН 171041003124)                  | ТОО «МедФарм» (БИН 990340005977)         | 96,1%    | 890 000 000                            |

🟠 **High** (доля 80-95%): 6 случаев (показаны топ-3)

| Заказчик                                                             | Поставщик                           | Доля |
| ---------------------------------------------------------------------------- | -------------------------------------------- | -------- |
| ГУ «Департамент образования» (БИН 231040023028) | ТОО «Билим-Сервис»           | 91,5%    |
| ГУ «Управление культуры» (БИН 120940001946)         | ТОО «КазМедиа»                  | 85,2%    |
| КГП «Центральная больница» (БИН 150540000186)     | ТОО «Фарм-Дистрибьюция» | 83,7%    |

**Метод оценки:** Расчёт доли каждого поставщика от общей суммы закупок заказчика: `share = SUM(supplier_contracts) / SUM(customer_total) × 100%`. Порог: 80%.

**Уверенность и ограничения:**

- Размер выборки: все 27 целевых организаций → уровень доверия **высокий**

- Концентрация может быть обоснованной (единственный поставщик в регионе, специализированные ТРУ)

**Рекомендации:**

1. Критические случаи (>95%) требуют приоритетной проверки — один поставщик практически монополизировал закупки.
2. Рекомендуется проверить, проводились ли конкурсные процедуры или закупки шли из одного источника.
3. Для организаций с долей >90% целесообразно проанализировать виды ТРУ — возможно, специализация поставщика оправдана.

## Пример 5: Аномалии объёмов (RU)

### Вопрос пользователя

> Есть ли аномальные объёмы закупок по сравнению с предыдущими годами?

### ReAct-трейс агента

```
Thought: Для выявления аномальных объёмов использую detect_volume_anomalies.
         Порог 2.0 — объём в 2+ раза больше среднего.
Action:  detect_volume_anomalies(threshold=2.0)
Observation: {total: 15, showing: 15, anomalies: [...]}
```

### Ответ агента

**Вердикт:** Обнаружено **15 случаев** аномального завышения объёмов закупок — объём за год превышает среднегодовой показатель в 2 и более раз. Максимальное превышение — в 8,3 раза.

- Период: 2024-2026 гг.

- Источники: анализ аномалий объёмов (тренды объёмов по годам)
- Размер выборки: данные из лотов со статусом «Завершён» (360)

2. Для аномалий с кратностью > 5× целесообразно запросить техническое задание и обоснование потребности.

## Пример 6: Ответ на казахском языке (KZ)

### Сұрақ (Вопрос)

> Келісімшарттар сомасы бойынша топ-3 тапсырысшыны көрсет

### ReAct-трейс агента

```
Thought: Пайдаланушы қазақша сұрақ қойды. COALESCE(s.name_kz, s.name_ru) қолданамын.
Action:  execute_sql("SELECT c.customer_bin,
         COALESCE(s.name_kz, s.name_ru) as customer_name,
         COUNT(*) as contract_count,
         SUM(c.contract_sum) as total_sum
         FROM contracts c
         LEFT JOIN subjects s ON c.customer_bin = s.bin
         WHERE c.contract_sum > 0
         GROUP BY c.customer_bin, s.name_kz, s.name_ru
         ORDER BY total_sum DESC LIMIT 3")
Observation: 3 rows returned
```

### Жауап (Ответ)

**Қорытынды:** 2024-2026 жж. кезеңінде келісімшарттар сомасы бойынша ең ірі 3 тапсырысшы жалпы 217,9 млрд теңгеге келісімшарт жасасқан, бұл жалпы сомасының 51,5%-ін құрайды.

**Пайдаланылған деректер:**

- Кезең: 2024-2026 жж.
- Сүзгілер: барлық тапсырысшылар (27 ұйым), сома > 0
- Деректер көзі: SQL сұрауы (келісімшарттар, ұйымдар)
- Іріктеме көлемі: 32 574 келісімшарт

**Талдау:**

| № | Тапсырысшы                                     | БИН       | Келісімшарт саны | Жалпы сома (тг) |
| -- | -------------------------------------------------------- | ------------ | ------------------------------- | -------------------------- |
| 1  | Астана қ. Полиция департаменті | 000740001307 | 4 521                           | 98 450 000 000             |
| 2  | №1 Қалалық клиникалық аурухана | 050740004819 | 3 876                           | 67 230 000 000             |
| 3  | ШҚО Білім басқармасы                   | 210240019348 | 2 934                           | 52 180 000 000             |

**Бағалау әдісі:** SQL біріктіру (GROUP BY customer_bin, SUM, COUNT) — сома бойынша кему ретімен сұрыптау.

**Сенімділік және шектеулер:**

- Іріктеме көлемі: 32 574 келісімшарт → сенімділік деңгейі **жоғары**
