# Перечень рисков и ограничений

## Обзор

Документ описывает выявленные риски, ограничения и стратегии их митигации для системы AI-анализа госзакупок. Выделил только важные риски, но нужно учитывать все, пока только важные выписал.

---

## Детальное описание рисков

### R01. Некорректные цены в API

**Описание**: Поле `price_per_unit` в `contract_subjects` иногда содержит значение `total_price` (общую сумму вместо цены за единицу). Это системная ошибка данных API OWS v3.

**Влияние**: Искажение медианных цен, ложные аномалии, некорректные вердикты Fair Price.

**Митигация** (реализовано):

- Фильтр качества данных: `price_per_unit <= total_price * 1.1` при `quantity > 1`
- Функция `_data_quality_filter()` в `anomaly_detector.py`
- Фильтр применяется во всех аналитических модулях

**Остаточный риск**: Средний — фильтр не покрывает случаи, когда `quantity = 1` (цена за единицу = общая сумма).

---

### R02. Отсутствие кода ENSTRU у лотов

**Описание**: Часть лотов не имеет кода ENSTRU (ТРУ) — ключевого классификатора для расчёта Fair Price и поиска аналогов.

**Влияние**: Невозможность расчёта Fair Price, отсутствие аналогов для сравнения.

**Митигация** (реализовано):

- Fallback-поиск ENSTRU по названию лота через `_resolve_enstru_by_name()` — ищет наиболее частый ENSTRU-код в `contract_subjects` с похожим названием (trigram similarity, расширение `pg_trgm`)
- Поле `enstru_source = "resolved_from_contract_subjects"` маркирует подобранные коды
- Рекомендация пользователю в ответе: «ENSTRU-код подобран автоматически»

**Остаточный риск**: Средний — trigram-поиск может подобрать неточный код для специфических товаров.

---

### R03. Пустые KATO (регион поставки)

**Описание**: Поле `plnPointKatoList` в API пусто для некоторых лотов, хотя портал отображает KATO.

**Влияние**: Невозможность регионального анализа и расчёта регионального коэффициента.

**Митигация** (реализовано):

- Fallback-источник: `Plans.PlansKato.refKatoCode` — добавлен в GraphQL-запрос и `data_loader.py`
- UPSERT-запрос обновлён: `delivery_kato` включён в `ON CONFLICT DO UPDATE SET`
- При отсутствии KATO: региональный коэффициент = 1.0, аналоги ищутся без фильтра по региону

**Остаточный риск**: Низкий — fallback покрывает большинство случаев.

## Ограничения системы

### Функциональные ограничения

| Ограничение              | Описание                                                                    |
| ----------------------------------- | ----------------------------------------------------------------------------------- |
| 8 итераций ReAct            | Максимум 8 циклов Reason→Act→Observe за один вопрос     |
| 20 tool calls на вопрос     | Лимит вызовов инструментов за одно сообщение |
| 15 пар в истории         | Sliding window — ранние сообщения сессии теряются     |
| Русский + Казахский | Только два языка интерфейса                                 |

### Аналитические ограничения

| Ограничение         | Описание                                                                                                     |
| ------------------------------ | -------------------------------------------------------------------------------------------------------------------- |
| Минимум 3 записи  | Fair Price не рассчитывается при <3 записях с ценами                                |
| Сезонность: 5/20     | Сезонный коэффициент требует мин. 5 записей в квартале и 20 всего |
| IQR threshold 1.5              | По умолчанию — умеренный порог; для экстремальных аномалий: 3.0    |
| Концентрация ≥80% | Порог по умолчанию — 80% от суммы закупок у одного поставщика        |
| Объёмы ×2.0             | Порог по умолчанию — объём в 2+ раза больше среднегодового            |

### Инфраструктурные ограничения

| Ограничение                    | Описание                                                                         |
| ----------------------------------------- | ---------------------------------------------------------------------------------------- |
| 1 экземпляр приложения | Нет горизонтального масштабирования (1 uvicorn process) |
| Нет кэширования             | Нет Redis/Memcached для кэширования ответов                      |
